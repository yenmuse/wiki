对于初学 NR 的人来说，在各种书籍 / 文章中涉及到跟物理层资源相关的描述中，会经常看到 Point A(常伴随其出现的还有 CRB0,

我们先来看下协议对 Point A 的定义，如下 (定义就是第一句，不过后面关于其从何得来 (obtained from) 的两点还是很有用，后面计算 Point A 时会详细讲到)。

----clip----

//38.211-4.4.4.2

Point A serves as a common reference point for resource block grids and is obtained from:

- offsetToPointA for a PCell downlink where offsetToPointA represents the frequency offset between point A and the lowest subcarrier of the lowest resource block, whichhas the subcarrier spacing provided by the higher-layer parameter subCarrierSpacingCommon and overlaps with the SS/PBCH block used by the UE for initial cell selection, expressed inunits of resource blocks assuming 15 kHz subcarrier spacing for FR1 and 60 kHz subcarrier spacing for FR2;

-absoluteFrequencyPointA for all other cases where absoluteFrequencyPointA represents the frequency-location of point A expressed as in ARFCN.

----clip----

这里有两个关键词：“common reference point” 和 “resource block grids”.

首先不同的 SCS(Sub-Carrier Spacing) 对应的 RB 和 slot 的 size 不同，因此不同 SCS 的有各自的 resource block grid, resource block grid 用 CRB– Common Resouce Block 表示，从 CRB0 开始。

不同的 numerology 的时频域形成嵌套式结构 (Nested Structure), 比如 SCS30 对应的 RB size 是 SCS15 的 2 倍 (而 slot 长度则是 SCS15 的 1/2)：

因此在同一 channel bandwidth(整个系统带宽，以 mHz 为单位) 下, transmission bandwidth(实际使用的带宽，以 RB 为单位) 也不同：

然而上表中，以 10mHz 系统带宽为例，虽然 SCS15 的 RB size 是 SCS 的一半，但对应的 RB 个数 (52) 并非 SCS30 时 (24) 的 2 倍，原因是，不同的 SCS 的系统带宽对应的 guand band 不同，这样不同 SCS 的对应的 transmission bandwidth 大小和物理位置都不一样，而由于 NR 引入 BWP 概念，站在 UE 的角度，其看到的世界就是 BWP 范围 ---- 整个 transmission bandwidth 带宽的一部分，每个 BWP 对应一个 SCS 下的一段实际物理带宽 (用 PRB 来表示)，那如何知道各 BWP 的起始位置？并且针对不同 SCS 的 resource block grid 能用统一的方式来计算呢？这就需要一个公共基准点 -- common reference point，这样就有了 Point A(当然，BWP 不是简单的以到 Point A 的距离计算，因为 bandwidth 不一定以 PointA 为起点，而是有个 OffsetToCarrier，不过这一篇主要讲 Point A 的计算，BWP 起始位置如何表示和计算会在 BWP 系列的 OffsetToCarrier&locationAndBandwidth 解析部分讲解))。

Point A 对于所有 SCS 的 resource block grid 来说都相同，其位置对应于每个 SCS 的资源网格的 CRB0 的第一个子载波 (Subcarrier0) 的中心位置, 这样每个 BWP 都可以根据 Point A 计算自己的起始位置。
